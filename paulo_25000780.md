-1. Regras de Negócio

1. Produtos só podem ser vendidos se houver quantidade disponível em estoque.  
2. O sistema deve impedir a venda de produtos sem estoque.  
3. Produtos abaixo do nível mínimo devem gerar alerta automático.  
4. Vendas a prazo devem ser registradas em contas a receber com controle de status.  
5. Compras devem atualizar automaticamente o estoque da unidade.  
6. Apenas farmacêuticos podem validar receitas médicas.  
7. Toda venda deve gerar um comprovante com os dados da operação.  

-2. Requisitos Funcionais

1. Cadastrar, editar e excluir produtos.  
2. Consultar produtos por nome, código de barras ou fabricante.  
3. Registrar vendas à vista e a prazo.  
4. Verificar disponibilidade de estoque automaticamente.  
5. Atualizar estoque após vendas, compras, devoluções ou perdas.  
6. Cadastrar clientes e manter histórico de compras.  
7. Registrar compras de fornecedores.  
8. Gerenciar contas a pagar.  
9. Gerenciar contas a receber.  
10. Emitir relatórios gerenciais.  
11. Gerenciar usuários e permissões.  

-3. Requisitos Não Funcionais

1. O sistema deve ser acessível via navegador web.  
2. O tempo de resposta deve ser inferior a 3 segundos.  
3. O sistema deve garantir autenticação e controle de acesso por usuário.  
4. O sistema deve ter disponibilidade mínima de 99%.  
5. A interface deve ser intuitiva e de fácil utilização.

-7. Documentação de Caso de Uso

### Caso de Uso: Realizar Venda

- Ator: Atendente  
- Descrição: Permite registrar a venda de produtos ao cliente.  

Pré-condições:
- Produto cadastrado  
- Produto com estoque disponível  

Fluxo Principal:
1. Atendente consulta produto  
2. Sistema exibe informações  
3. Atendente informa quantidade  
4. Sistema verifica estoque  
5. Sistema registra venda  
6. Sistema emite comprovante  

Fluxos Alternativos:
- Produto sem estoque → sistema bloqueia a venda  
- Cliente não cadastrado → sistema permite cadastro
